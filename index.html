PROJECT_01 · prompt-as-asset

Harness 范式 · Simple Template

角色：方案设计 + 落地 · 通用 AI 平台组
自动化链路
Problem

CCR 周度归因高度依赖人肉调试：每次跑一遍要 8 小时，且因为 prompt 不稳定、上下文易飘，单次成功率不到 30%。分析师在「调 prompt」这件事上反复消耗，结论无法沉淀、无法复用。

Approach
  • 用 Harness 范式把 Prompt 工程化，做成参数化、可复用的 Template，让能力变成可被调用的资产
  • 固化输入/输出契约与执行边界，把「调试」收敛成「填参数 + 跑模板」
  • 结合定时任务编排自动化链路，让归因报告每周自动产出，无需人工介入

PROJECT_02 · multi-tenant platform

Thunderbolt Agent 平台 · Skill Hub

角色：平台产品设计 · 面向全业务线
AI 原生平台
Problem

业务方都想用 AI，但每个团队各搭各的、重复造轮子，能力散落、无法治理。缺一个让非技术同学也能低门槛搭建、管理自己 Agent 的统一底座。

Approach
  • 设计内部多租户 Agent 平台，业务方可自助搭建、配置、管理专属 Agent
  • 沉淀 Skill Hub：把能力抽象成可挂载、可组合的 Skill，跨团队复用
  • 统一编排与权限层，让「搭 Agent」从工程活变成产品配置动作

PROJECT_03 · content governance

CCR 内容合规质检

角色：AI 方案落地 · 多站点合规
生产级 · 26 国
Problem

TikTok Shop 跨 26 国站点的内容合规打标，规则多、语言多、量级大，纯人工无法覆盖，且各站点标准不一致，质量难以横向对齐。

Approach
  • 用大模型做内容合规自动打标，覆盖多语言、多站点的巡检场景
  • 把合规规则结构化为模型可执行的判定标准，统一各站点口径
  • 持续用真实数据回流校准，把准召稳定在生产可用水位

PROJECT_04 · model self-check

Auto-QA 模型自检工具

角色：质检范式设计 · 模型自驱
自进化
Problem

质检长期靠人肉抽检，覆盖率低、滞后，系统性的 gap 往往要等出事才被发现，无法主动暴露模型的盲区。

Approach
  • 让模型「自己检查自己」，把质检从被动抽检升级为模型自驱的主动巡检
  • 设计自检范式，主动发现系统性 gap 与高失败模式，给出可定位的洞察
  • 把自检结果回流到迭代闭环，形成「发现问题 → 修复 → 再自检」的循环
